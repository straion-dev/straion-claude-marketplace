# Terminology

| Term            | Definition                                                                                  |
| --------------- | ------------------------------------------------------------------------------------------- |
| **Spec**        | Specification document (PRD, ADR, RFC, or lightweight feature spec), or implementation plan |
| **Task**        | Single implementation unit with clear acceptance criteria                                   |
| **Requirement** | Project requirement from Straion that code/specs must comply with                           |
| **Validation**  | Checking compliance against requirements                                                    |
| **Compliance**  | Meeting requirement constraints (✓ Compliant, ⚠ Partial, ✗ Violation)                       |
